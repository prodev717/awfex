API Endpoints

POST /workflow
Saves or updates a workflow.
{
  "name": "mathFlow",
  "workflow": { "mul": [{ "add": [2, 3] }, 4] }
}
GET /workflow/:name
Retrieve workflow JSON by name.
GET /workflow
List all workflow names.
DELETE /workflow/:name
Deletes a workflow by name.
DELETE /workflow/userSignupFlow
GET /run/:name
Runs a workflow stored in the database.
Supports query parameters and environment values. 
GET /run/mathFlow?x=10&y=20
POST /run
Runs a workflow sent directly in the POST body.
Accepts query parameters and environment lookups. 
POST /run?x=5
{
  "add": ["$query:x", 3]
}
GET /functions
Returns a list of available functions.

Special Value Types

1. Using Query Parameters — $query:

You can inject query string values into any workflow.
"$query:key"
Example: 
{
  "add": ["$query:a", "$query:b"]
}

GET /run? a=7 & b=5
→ 12

2. Using Environment Variables — $env:

Reads values directly from your .env file using: 
"$env:VARIABLE_NAME"
Example .env: 
SECRET_KEY=12345
Example Workflow: 
{
  "print": ["$env:SECRET_KEY"]
}
Output: 
12345
Values are substituted at runtime using process.env.

3. Number Auto-Detection

Query parameters like ?x=10 automatically become numbers.
Non-numeric values remain strings: 
{
  "add": ["$query:x", 5]
}

GET /run?x=10 → 15
GET /run?x=hello → "hello5" (string concatenation)
